# 💻 AGENT_GUIDE.md (คู่มือการทำงานของพนักงาน AI)

## 🎯 ปรัชญาการทำงานของ AI Dev (Ponytail Spirit)
*   **"โค้ดที่ดีที่สุด คือโค้ดที่ไม่ต้องเขียน"**
*   ห้ามสร้างไฟล์หรือเขียนโปรแกรมเผื่ออนาคต
*   พยายามเขียนโค้ดให้อ่านง่าย กระชับ และไม่สร้างภาระให้ AI ตัวถัดไปต้องอ่านเยอะ (ช่วยเซฟโทเคนบอส)

## 🛠️ โครงสร้างของโปรเจคย่อย (Sub-Apps Breakdown)
เมื่อบอสเพิ่มโปรเจคอื่นๆ เข้ามา ให้แบ่งย่อยดังนี้:
1. `apps/dashboard/`: หน้าเว็บ Office Portal (Vite + React)
2. `apps/worker-api/`: ส่วนประสานงาน APIs (Cloudflare Workers)
3. `apps/local-worker/`: รันสคริปต์บน VPS/เครื่องบอส (Playwright/Node)

## 📌 ขั้นตอนส่งงาน (Pull Request Checklist)
ก่อนส่งมอบงานให้บอส AI จะต้อง:
- [ ] เช็คว่าไม่มี error ลินต์และคอมไพล์ (`npm run build` ผ่าน)
- [ ] โค้ดทั้งหมดทำตามบันไดการตัดสินใจ Ponytail แล้ว
- [ ] มีการล็อกสถานะและบันทึก logs ลงระบบชัดเจน
